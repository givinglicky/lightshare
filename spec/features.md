## 2. 詳細功能規格

### 2.1 認證模組

#### 2.1.1 使用者註冊

**功能描述**: 新用戶建立帳號

**需求規格**:
- 必填欄位: 電子郵件、密碼、使用者名稱
- 密碼強度要求: 至少 6 個字元
- 電子郵件格式驗證
- 使用者名稱唯一性檢查

**UI 規格**:
- 註冊表單包含: 姓名、Email、密碼、確認密碼
- 密碼可視/隱藏切換
- 註冊成功後自動登入並跳轉至 feed

#### 2.1.2 使用者登入

**功能描述**: 現有用戶登入系統

**需求規格**:
- 支援 Email + 密碼登入
- 保持登入狀態 (JWT token)
- 登入失敗顯示錯誤訊息

**UI 規格**:
- 登入表單包含: Email、密碼
- 忘記密碼連結
- 註冊新帳號連結

### 2.2 貼文模組

#### 2.2.1 發佈貼文

**功能描述**: 發起求助或分享故事

**需求規格**:
- 貼文類型: 求助分享、故事分享
- 必填欄位: 標題、內容
- 支援分類標籤
- 發佈前可預覽

**UI 規格**:
- 3 步驟發佈流程:
  1. 選擇類型 (求助分享 / 故事分享)
  2. 編輯內容
  3. 預覽並發佈

#### 2.2.2 瀏覽動態 (Feed)

**功能描述**: 查看社群最新貼文

**需求規格**:
- 按時間倒序顯示
- 無限滾動載入
- 顯示貼文摘要 (標題 + 部分內容)

**UI 規格**:
- 置頂導航栏
- 卡片式貼文列表
- 浮動發佈按鈕

#### 2.2.3 貼文詳情

**功能描述**: 查看完整貼文內容與互動

**需求規格**:
- 顯示完整貼文內容
- 顯示按讚數、留言數
- 顯示發佈者資訊
- 支援收藏功能

**UI 規格**:
- 麵包屑導航
- 詳細內容區塊
- 互動按鈕列 (按讚、收藏、分享)
- 留言區塊

### 2.3 互動模組

#### 2.3.1 按讚支持

**功能描述**: 對貼文表示支持

**需求規格**:
- 點擊切換 liked 狀態
- 即時更新按讚數
- 不能對自己貼文按讚

**UI 規格**:
- 愛心圖標
- 顯示按讚數
- liked 狀態顯示紅色愛心

#### 2.3.2 收藏貼文

**功能描述**: 收藏有價值的貼文

**需求規格**:
- 點擊切換 bookmarked 狀態
- 在個人檔案可查看所有收藏

**UI 規格**:
- 書籌圖標
- bookmarked 狀態顯示填滿書籌

#### 2.3.3 留言回應

**功能描述**: 在貼文下發表留言

**需求規格**:
- 文字留言
- 顯示留言時間
- 顯示留言者資訊

**UI 規格**:
- 輸入框 + 發送按鈕
- 留言列表 (時間倒序)

### 2.4 使用者模組

#### 2.4.1 個人檔案

**功能描述**: 展示用戶資訊與發佈內容

**需求規格**:
- 顯示用戶名稱、頭像、bio
- 顯示用戶發佈的貼文
- 顯示用戶收藏的貼文

**UI 規格**:
- 個人資訊卡片
- 分頁切換: 我的貼文 / 收藏

#### 2.4.2 編輯資料

**功能描述**: 修改個人檔案資訊

**需求規格**:
- 可編輯: 名稱、bio
- 頭像上傳 (待實現)

### 2.5 通知模組

#### 2.5.1 通知列表

**功能描述**: 查看所有系統通知

**需求規格**:
- 通知類型: 按讚、收藏、留言、新追蹤
- 按時間倒序顯示
- 標記已讀/未讀

#### 2.5.2 通知設定

**功能描述**: 客製化通知偏好

**需求規格**:
- 接收按讚通知 (開/關)
- 接收收藏通知 (開/關)
- 接收留言通知 (開/關)
- 接收新追蹤通知 (開/關)

### 2.6 搜尋模組

#### 2.6.1 搜尋貼文

**功能描述**: 關鍵字搜尋貼文

**需求規格**:
- 支援標題關鍵字搜尋
- 支援內容關鍵字搜尋
- 顯示搜尋結果數量

## 3. 非功能需求

### 3.1 效能需求

- 首頁載入時間: < 3 秒
- API 回應時間: < 500ms
- 支援同時 100+ 使用者

### 3.2 安全需求

- 密碼加密儲存 (bcrypt)
- JWT token 驗證
- 輸入內容 sanitization
- XSS 攻擊防護

### 3.3 可用性需求

- 支援主流瀏覽器 (Chrome, Firefox, Safari, Edge)
- 響應式設計 (手機/平板/桌面)
- 無障礙支援 (基本)

### 3.4 擴展性需求

- 模組化架構
- 支援未來功能擴展
- API RESTful 設計
